We created this project to turn our real-life AR/VR/MR classroom into a virtual one. It uses Unity to build the room and MetaHumans to represent the people inside, making the digital environment look and feel just like the real thing.

The Process:
To build this application, we started by measuring and modeling our physical AR/VR/MR classroom to recreate it accurately in the Unity game engine. The project is organized into clear sections: Scenes for the room layout, Prefabs for reusable objects like furniture, and C# Scripts to handle how users move and interact. We used the Meta XR SDK to ensure the app works perfectly with Quest headsets and the Universal Render Pipeline to keep the visuals high-quality without slowing down the performance. To make the space feel more natural, we integrated MetaHumans to represent the people in the room.
To run the project, open the folder in Unity and ensure you have the necessary Meta packages installed. Once you have your Meta Quest headset connected, you can use the Play button in the editor for a quick test or build an APK file to install the application directly onto the device. This setup allows anyone with the right hardware to step into our virtual classroom and experience the digital environment firsthand.


Setbacks-
Building this project came with several hurdles, primarily involving the Meta Quest hardware. For a large portion of the development time, the headset was not working correctly, which made it very difficult to test features in real-time. We ran into numerous technical issues with the initial setup, from connection bugs to software version mismatches, which slowed down our progress. These challenges taught us a lot about troubleshooting hardware-software integration and the importance of having a solid testing plan when working with emerging tech like VR.
Because of these early setbacks, there wasn't quite enough time to build everything we originally envisioned. In the future, we hope to expand the project by adding more interactive classroom elements, such as a working whiteboard or the ability for multiple users to join the same session. We also plan to improve the MetaHuman animations to make the virtual students look even more lifelike. Reflecting on this experience, we've learned how to better manage technical roadblocks and will approach future projects with a more flexible schedule for hardware setup and debugging.
